# Data Availability

This repository provides source code, documentation, one fixed auxiliary input file required by the full reproducibility pipeline, and an application-ready trained FADS_AI model.

Precomputed intermediate outputs from the main Monte Carlo experiment are not distributed.

## Distributed files

The repository distributes the source code, documentation, and the following fixed auxiliary input file required by the full pipeline:

```text
data/input_data/poligonos_pre.rds
```

This file contains preprocessed L-moment Ratio Diagram (LMRD) polygon information required to compute LMRD-based features.

This file is distributed because it is a fixed auxiliary input, not a generated output of the Monte Carlo experiment.

## Application-ready trained model

The final trained model intended for application is distributed with the release:

```text
models/fads_ai_final_model.rds
```

The application model is distributed together with metadata required to apply it to new annual-maximum samples:

```text
models/fads_ai_feature_columns.rds
models/fads_ai_model_metadata.json
models/fads_ai_feature_dictionary.csv
models/fads_ai_candidate_distributions.csv
```

The released application model uses the classical L-moment feature scenario and the XGBoost classifier:

```text
scenario  = classical
algorithm = xgb
features  = lmom_l1, lmom_l2, lmom_l3, lmom_l4, lmom_t3, lmom_t4
```

This application-ready model is distinct from intermediate training artifacts generated during the reproducibility pipeline.

## Files not distributed

The following products are not distributed as static files:

```text
raw Monte Carlo samples
simulation metadata generated by the pipeline
feature tables
goodness-of-fit and information-criterion tables
machine-learning datasets
intermediate trained-model markers from pipeline runs
validation predictions
test predictions
evaluation summaries
manuscript tables
manuscript figures
run logs
temporary shards
intermediate chunks
```

These files are generated locally when the user runs the reproducibility pipeline.

## Raw simulated samples

Raw Monte Carlo samples are not stored as static files.

Instead, samples are regenerated deterministically from:

```text
distribution family
parameter values
sample size
replicate identifier
random seed
simulation grid
```

This design reduces storage requirements while preserving computational reproducibility.

## Generated outputs

Generated outputs are written under:

```text
outputs/
```

Typical generated folders include:

```text
outputs/config/
outputs/simulated_samples/
outputs/feature_tables/
outputs/gof_tables/
outputs/datasets/
outputs/ml_models/
outputs/evaluation/
outputs/run_logs/
outputs/paper_output/
```

These folders are intentionally excluded from version control.

## Reproducing the intermediate outputs

To regenerate the main computational outputs, run the full pipeline from the repository root:

```r
source("scripts/900_run_full_pipeline.R")
```

To force a clean run from the beginning:

```r
options(fads.run_mode = "clean")
source("scripts/900_run_full_pipeline.R")
```

The full experiment may be computationally demanding. Users should inspect:

```text
scripts/000_config.R
```

before running the complete workflow.

## Application example

The application example is provided under:

```text
examples/
```

It uses the trained model distributed in:

```text
models/fads_ai_final_model.rds
```

and does not require users to rerun the full Monte Carlo experiment.

The example input file must contain a numeric column named:

```text
value
```

representing the annual maximum value of the variable being analyzed.
