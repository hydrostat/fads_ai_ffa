# Reproducibility Notes

This document describes the reproducibility logic of **FADS_AI** (*Flood-frequency Analysis Distribution Selection with Artificial Intelligence*).

The repository is designed so that users regenerate the main computational outputs from source code rather than downloading precomputed experiment outputs.

## General policy

Precomputed outputs from the main Monte Carlo experiment are not distributed with this repository.

The following products are generated locally when the pipeline is executed:

```text
simulation metadata
feature tables
goodness-of-fit and information-criterion tables
machine-learning datasets
trained-model markers
validation outputs
test-evaluation outputs
manuscript tables
manuscript figures
```

Raw Monte Carlo samples are not stored as static files. Samples are regenerated deterministically from the simulation grid, distribution family, parameter values, sample size, and random seed.

## Repository root

All scripts should be run from the root folder of the repository:

```text
fads_ai_ffa/
```

In R, set the working directory to the repository root before running scripts:

```r
setwd("path/to/fads_ai_ffa")
```

The scripts check for the expected repository structure before execution.

## Required fixed input

The pipeline requires one fixed auxiliary input file:

```text
data/input_data/poligonos_pre.rds
```

This file contains preprocessed L-moment Ratio Diagram polygon information used to compute LMRD-based features.

It is distributed with the repository because it is a fixed input required by the feature-extraction stage, not a generated output of the Monte Carlo experiment.

## Main pipeline

The full pipeline is executed by:

```r
source("scripts/900_run_full_pipeline.R")
```

The main workflow is:

```text
001_create_project_structure.R
000_config.R
010_simulate_samples.R
020_extract_features.R
021_extract_gof_features.R
022_combine_gof_chunks.R
030_build_datasets.R
040_train_models.R
050_evaluate_models.R
```

## Step descriptions

### `001_create_project_structure.R`

Verifies the public repository structure and creates generated-output folders under:

```text
outputs/
```

It also checks that the fixed LMRD input file exists.

### `000_config.R`

Defines the central configuration object and saves it to:

```text
outputs/config/config_main.rds
```

The configuration includes:

```text
candidate distributions
sample sizes
number of Monte Carlo replicates
random seed
train/validation/test proportions
active machine-learning models
feature-extraction settings
GOF/IC chunk settings
output paths
```

The public configuration should be inspected before running the full experiment, especially the Monte Carlo size and parallel-processing settings.

### `010_simulate_samples.R`

Builds the candidate-distribution parameter registry, applies population-specific positive-domain shifts, validates parameter sets, and creates the simulation grid.

This step saves simulation metadata but does not store raw simulated samples.

### `020_extract_features.R`

Regenerates each sample from the simulation metadata and computes:

```text
sample L-moments
L-moment ratios
LMRD membership features
LMRD acceptance-count features
```

Feature tables are written by true distribution family.

### `021_extract_gof_features.R`

Regenerates each sample and computes fitted-distribution diagnostics for all candidate families.

The outputs include:

```text
Kolmogorov-Smirnov statistics and approximate p-values
Anderson-Darling statistics and approximate p-values
Cramer-von Mises statistics and approximate p-values
log-likelihood
AIC
BIC
minimum Bayes-factor-type descriptors derived from GOF p-values
```

The EDF-based p-values are approximate because distribution parameters are estimated from the same sample.

### `022_combine_gof_chunks.R`

Combines the GOF/IC chunk files generated by step `021` into family-level GOF tables.

### `030_build_datasets.R`

Joins feature tables and GOF/IC tables by `sample_id`, defines feature scenarios, and creates training, validation, and test datasets.

The split is performed within each true distribution family using deterministic family-specific seeds and the proportions defined in `scripts/000_config.R`.

### `040_train_models.R`

Trains the active machine-learning models defined in the configuration file across the feature scenarios.

Model selection is based on validation-set performance.

For storage efficiency, this script saves minimal model markers for selected model-scenario combinations rather than full fitted-model objects.

### `050_evaluate_models.R`

Evaluates validation-selected model-scenario combinations on the held-out test dataset.

The test set is not used for model selection. It is used only for final evaluation.

If minimal model markers are detected, the selected models are retrained in memory before test-set evaluation.

## Running modes

The default execution mode is resume mode:

```r
source("scripts/900_run_full_pipeline.R")
```

In resume mode, completed steps are skipped when their expected outputs are detected.

To force a clean run from the beginning:

```r
options(fads.run_mode = "clean")
source("scripts/900_run_full_pipeline.R")
```

To force rerun of a specific step:

```r
options(fads.force_steps = c("040_train_models"))
source("scripts/900_run_full_pipeline.R")
```

To clean outputs for a specific step before running:

```r
options(fads.clean_steps = c("040_train_models"))
source("scripts/900_run_full_pipeline.R")
```

## Generated-output folders

Generated outputs are written under:

```text
outputs/
```

Typical generated folders include:

```text
outputs/config/
outputs/simulated_samples/
outputs/feature_tables/
outputs/gof_tables/
outputs/datasets/
outputs/ml_models/
outputs/evaluation/
outputs/run_logs/
outputs/paper_output/
```

These folders are not intended to be committed to GitHub.

## Computational cost

The full Monte Carlo experiment may be computationally demanding.

Before running the full workflow, users should inspect:

```text
scripts/000_config.R
```

In particular, users may adjust:

```text
n_rep_mc
sample_sizes
features$chunk_size
gof$chunk_size
gof$parallel
gof$workers
ml$models
```

Changing these settings changes the generated outputs and may make results differ from those reported in the associated manuscript.

## Reproducibility checks

The repository includes lightweight checks under:

```text
tests/
```

For example:

```r
source("tests/check_repository_integrity.R")
```

This check verifies:

```text
expected folder structure
required files
syntax of R scripts
presence and structure of the fixed LMRD input object
availability of required R packages
```

The integrity check does not run the full Monte Carlo experiment.

## Notes on interpretation

FADS_AI is intended as a decision-support framework for probability-distribution selection in FFA.

The machine-learning classifiers are trained and evaluated under controlled simulation conditions in which the generating distribution is known. In real applications, the true generating distribution is unknown, so model predictions should be interpreted as support for candidate families rather than definitive identification of a true distribution.

Goodness-of-fit p-values and derived Bayes-factor-type descriptors should also be interpreted cautiously, because the fitted parameters are estimated from the same sample used for the GOF calculations.

## Versioned release

The public GitHub repository is intended for code access and usability.

A stable version corresponding to the manuscript should be archived through Zenodo or a similar repository and assigned a permanent DOI.
